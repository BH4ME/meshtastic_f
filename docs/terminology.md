# 术语表

> 文档定位：本文档用于统一仓库中常见术语的含义，便于阅读各专题文档时保持同一语义。

## 1. 核心术语

| 术语 | 含义 |
| --- | --- |
| `board` | 具体硬件板型或设备定义，通常对应 `boards/` 下的 JSON 文件。 |
| `variant` | 某个板子的硬件适配实现，通常位于 `variants/<platform>/<board>/`。 |
| `module` | 功能模块，负责单一能力或单一子系统，例如路由、遥测、通知。 |
| `PowerFSM` | 电源状态机，负责睡眠、供电和唤醒策略。 |
| `main.cpp` | 固件主入口，负责启动编排和主循环。 |
| `setupModules()` | 模块装配入口，负责创建和注册功能模块。 |
| `loop()` | 主循环入口，负责持续调度和轻量轮询。 |
| `routing` | 消息分发与转发逻辑，是消息处理链路的核心。 |
| `telemetry` | 遥测数据采集与上报能力。 |
| `I2C scan` | 启动时扫描 I2C 外设，用于探测屏幕、RTC、传感器等设备。 |
| `screen` | 显示与交互设备，可能是 TFT、E-Ink 或其他屏幕。 |
| `status object` | 跨模块共享的状态容器，例如电源、GPS、蓝牙状态。 |

## 2. 运行术语

| 术语 | 含义 |
| --- | --- |
| `SDS` | 深度睡眠。 |
| `LS` | 轻睡眠。 |
| `NB` | 正常运行模式。 |
| `POWER` | 外部供电运行模式。 |
| `SERIAL` | 串口优先交互状态。 |
| `DARK` | 屏幕关闭但系统仍可交互的状态。 |

## 3. 角色与边界

| 术语 | 含义 |
| --- | --- |
| `input side` | 按键、串口、蓝牙、无线输入等进入系统的入口。 |
| `output side` | 屏幕、串口输出、通知、LED 和蜂鸣器等系统输出。 |
| `core protocol` | 路由、消息格式和协议处理逻辑。 |
| `platform layer` | 与平台相关的底层差异实现。 |
| `cross-platform logic` | 可跨多个平台复用的通用逻辑。 |
